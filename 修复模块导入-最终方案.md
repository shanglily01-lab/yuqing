# 修复模块导入问题 - 最终方案

## 问题：ModuleNotFoundError: No module named 'config'

即使设置了PYTHONPATH，仍然无法导入config模块。

## 解决方案

### 方法1：检查config.py是否存在

```bash
cd ~/yuqing
ls -la config.py
```

如果文件不存在，需要创建它或从config.py.example复制。

### 方法2：使用绝对路径设置PYTHONPATH

```bash
cd ~/yuqing
export PYTHONPATH=/home/ec2-user/yuqing:$PYTHONPATH
cd MindSpider
python main.py --setup
```

### 方法3：直接在项目根目录运行

```bash
cd ~/yuqing
export PYTHONPATH=/home/ec2-user/yuqing:$PYTHONPATH
python MindSpider/main.py --setup
```

### 方法4：使用sys.path.insert（临时修改代码）

如果以上都不行，可以临时修改MindSpider/main.py，在导入config之前添加：

```python
import sys
from pathlib import Path
project_root = Path(__file__).parent.parent.parent
sys.path.insert(0, str(project_root))
```

### 方法5：直接运行schema下的初始化脚本

```bash
cd ~/yuqing
source venv/bin/activate
export PYTHONPATH=/home/ec2-user/yuqing:$PYTHONPATH
cd MindSpider/schema
python init_database.py
```

## 验证PYTHONPATH

```bash
cd ~/yuqing
export PYTHONPATH=/home/ec2-user/yuqing:$PYTHONPATH
python -c "import sys; print('\n'.join(sys.path))"
python -c "import config; print('config imported successfully')"
```

## 检查config.py

```bash
cd ~/yuqing
# 检查文件是否存在
ls -la config.py

# 如果不存在，检查是否有示例文件
ls -la config.py.example

# 如果只有示例文件，复制它
cp config.py.example config.py
```
