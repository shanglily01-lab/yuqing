# 如何拉取 GitHub 仓库到本地

## 方式1：使用 SSH（推荐）

如果您已经配置了SSH密钥，可以使用以下命令：

```bash
# 克隆仓库
git clone git@github.com:shanglily01-lab/yuqing.git

# 进入项目目录
cd yuqing
```

## 方式2：使用 HTTPS

如果还没有配置SSH密钥，可以使用HTTPS方式：

```bash
# 克隆仓库
git clone https://github.com/shanglily01-lab/yuqing.git

# 进入项目目录
cd yuqing
```

## 方式3：如果本地已有项目，需要更新

如果本地已经有这个项目，想要拉取最新代码：

```bash
# 进入项目目录
cd yuqing

# 拉取最新代码
git pull origin main
```

## 首次克隆后的设置

克隆完成后，建议进行以下配置：

### 1. 检查项目结构
```bash
ls -la
```

### 2. 查看分支
```bash
git branch -a
```

### 3. 配置环境（如果需要）
```bash
# 复制配置文件模板
cp .env.example .env

# 编辑配置文件
vi .env  # 或使用其他编辑器
```

## 完整克隆示例

```bash
# 1. 克隆仓库
git clone git@github.com:shanglily01-lab/yuqing.git

# 2. 进入项目目录
cd yuqing

# 3. 查看项目结构
ls -la

# 4. 查看提交历史
git log --oneline -10

# 5. 查看当前分支
git branch
```

## 注意事项

1. **SSH密钥配置**：如果使用SSH方式，需要先配置SSH密钥
   - 检查是否已有密钥：`ls -al ~/.ssh`
   - 如果没有，生成新密钥：`ssh-keygen -t ed25519 -C "your_email@example.com"`
   - 将公钥添加到GitHub账户

2. **HTTPS认证**：如果使用HTTPS，可能需要输入GitHub用户名和Personal Access Token

3. **项目依赖**：克隆后需要安装依赖
   ```bash
   # 创建虚拟环境
   python3 -m venv venv
   source venv/bin/activate  # Linux/Mac
   # 或
   venv\Scripts\activate  # Windows
   
   # 安装依赖
   pip install -r requirements.txt
   ```

## 常见问题

### Q: 提示权限被拒绝（Permission denied）
A: 检查SSH密钥是否正确配置，或使用HTTPS方式

### Q: 提示仓库不存在
A: 确认仓库URL是否正确，仓库是否为公开仓库或您有访问权限

### Q: 克隆速度慢
A: 可以尝试使用GitHub镜像或配置代理
