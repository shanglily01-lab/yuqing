# 快速修复 - 模块导入问题

## 问题：ModuleNotFoundError: No module named 'config'

这是因为 `MindSpider` 需要从项目根目录导入 `config` 模块。

## 解决方案

### 方法1：从项目根目录运行（推荐）

```bash
# 确保在项目根目录
cd ~/yuqing

# 设置PYTHONPATH
export PYTHONPATH=$(pwd):$PYTHONPATH

# 激活虚拟环境（如果存在）
if [ -d "venv" ]; then
    source venv/bin/activate
fi

# 进入MindSpider目录并运行
cd MindSpider
python3 main.py --setup
cd ..
```

### 方法2：直接使用Python运行（最简单）

```bash
# 在项目根目录
cd ~/yuqing

# 设置环境变量并运行
PYTHONPATH=$(pwd) python3 MindSpider/main.py --setup
```

### 方法3：使用修复后的脚本

我已经修复了 `setup_database.sh` 脚本，现在可以这样运行：

```bash
cd ~/yuqing
./deploy/setup_database.sh
# 当提示项目路径时，输入：/home/ec2-user/yuqing
```

### 方法4：直接运行初始化脚本

```bash
cd ~/yuqing

# 设置PYTHONPATH
export PYTHONPATH=$(pwd):$PYTHONPATH

# 激活虚拟环境（如果存在）
if [ -d "venv" ]; then
    source venv/bin/activate
fi

# 直接运行数据库初始化脚本
cd MindSpider/schema
python3 init_database.py
cd ../..
```

## 验证修复

运行后应该看到类似输出：
```
[INFO] 数据表与视图创建完成
```

## 如果还有问题

检查以下几点：

1. **确认config.py存在**
   ```bash
   ls -la ~/yuqing/config.py
   ```

2. **确认.env文件存在**
   ```bash
   ls -la ~/yuqing/.env
   ```

3. **检查Python路径**
   ```bash
   cd ~/yuqing
   python3 -c "import sys; sys.path.insert(0, '.'); import config; print('OK')"
   ```
